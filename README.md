# Tasks App

Simple Spring Boot REST API for managing tasks. The application stores tasks in memory, exposes endpoints to list tasks, create a task, and mark a task as done, and returns structured error responses through `TaskExceptionHandler`.

## Getting started

### Requirements

- Java 26
- Maven 4 compatible build environment

### Run locally

From the project root:

```bash
mvn spring-boot:run
```

The application starts on the default Spring Boot port, `8080`.

### Build and run the JAR

```bash
mvn clean package
java -jar target/tasks-app-1.0-SNAPSHOT.jar
```

## API

- `GET /tasks` - returns all tasks
- `POST /tasks` - creates a task from a JSON body
- `PUT /tasks/{id}/done` - marks a task as done
- Postman collection: [docs/postman/tasks-app.json](docs/postman/tasks-app.json)

### Example request

```json
{
  "title": "Prepare interview task",
  "description": "Finish the task app implementation",
  "done": false
}
```

## Known limitations

- Data is stored in an in-memory `HashMap`, so all tasks are lost when the application stops.
- The repository is not thread-safe, so concurrent writes are not protected.
- Task IDs are generated from the current collection size, which is only suitable for this simple in-memory implementation.
- There is no persistence layer, authentication, pagination, or request validation.
- The API currently supports create, list, and mark-done operations only.

## Potential future extensions

- Replace the in-memory repository with a persistent store such as PostgreSQL, MySQL, or H2.
- Add full CRUD support, including update and delete endpoints.
- Add request validation for task title and description.
- Add filtering, sorting, and pagination for task lists.
- Add test coverage for controller, service, mapper, and exception handling behavior.
- Add authentication and authorization if the API becomes user-facing.
